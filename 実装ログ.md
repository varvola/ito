## セッションログ

### Day 12（0719）— NPC購入リアリティ（しおかぜ特急車内）

*体験*
- たむろ場の常連が **4人** に増えた（マスター／ソラ／**ミオ**／**カイ**）
- 「一緒に服屋へ」が **いつも買う** ではなくなった — しっくりこなければ見送る
- 買ったときの一言が、色・柄・滞在・道のり・空模様など **惹かれた理由** で変わる
- 見送ったときも、なぜ今日はやめたかの短い一言がある（責めないトーン）
- **ひとりで服屋に寄る日**がある（たむろ場を開いたとき、未着用の人が確率で買う）
- **話しかける**と服の話になり、会話が弾む選択肢だと買いやすくなる（「今日は別の話」なら買わない）
- **試着→印象を問う→反応で買う／買わない**（キャラ×親密度で質問が変わる・計96問）
- **親密度**は会話・（一緒の）購入で上がり、下がらない。ひとり買いは上がらない
- 親密度で口調が親しくなり、カイは口数が増え、ミオ等は恋愛錯覚ラインも出る

*実施（v39〜v41）*
- ミオ／カイ追加、スコア多要因、不買・理由台詞（v39）
- 日次自動来店・購入の原型
- 試着印象＋親密度（v41）

*判定（v41時点）*
- 購入FBは弱い／店とたむろの役割が混線 → v42 で分離へ

### Day 12（0719・続き）— 店とたむろの役割分離

*体験*
- **たむろ場** = 人がいたりいなかったりする交流の場。話すと服屋に来やすくなる。購入はしない
- **服屋** = 販売の場。来客に試着してもらい、印象の返事で買う／見送る。バッジで「販売中／〇〇が購入」
- たむろでは購入した服を着ている日も、**私服の日**もある
- ひとりで店に来て買う日もある（親密度は上がらない）
- 店頭が **販売中／購入済** の二段表示になり、買ったかが分かりやすい（v43）

*実施（v42〜v43）*
- `shopInterest`・日次 presence / wearing / shopVisitors
- たむろから「服屋の話」→ 来客リストへ。試着・購入は服屋のみ
- 購入フラッシュ＋販売中／購入済セクション分割

*判定*
- 役割分離の骨格 ✅。購入の視認性は v43 で改善
- 会話が「服を買うギミック」に見えた → v44 で生活ビートへ

### Day 12（0719・続き）— 用事のついで会話（生活ビート）

*体験*
- たむろの会話が **服の話から入らない**。キャラごとにその日の用事・気分がある（仕入れの帰り／遠回り／花を見てきた／本を返した など）
- カードに「仕入れの帰り」などの **今日のタグ** が出る
- 話を続けるうちに、自然に店が気になる選択が出る（押し売りCTAではない）
- 店に寄らなくても会話は完結する（用事の話だけで終わる枝もある）

*実施（v44）*
- `ITO_LIFE_BEATS` — NPC×3ビート、分岐ノード＋`nudge_shop`
- 日次 `meta.npcBeats`、たむろ主会話を life beat に差替え
- 「服屋の話をする」→「もう少し、用事の続きを」

*判定*
- 購入ギミック感は緩和。まだ短い木なので、実機で「生きてる」感を見て枝を足す
- 「誰が」は未達 → v45 で天候・気温・親密度の香辛料

### Day 12（0719・続き）— 天候・気温・親密度の会話差分

*体験*
- 同じ用事でも、**晴れ／雨／気温**と**親密度**で冒頭の一言が変わる（キャラ口調別）
- 雨の日は「雨宿り」など、空に寄せた用事が選ばれやすい
- カードに「仕入れの帰り · 雨 · 顔見知り」のように天気タグが出る

*実施（v45）*
- `ITO_WEATHER_SPICE`（性格×天気/気温×親密度3段）
- ライブ天気／キャッシュをたむろ会話に接続、ビート重み付け

*判定*
- GPSゲームらしい「今日ここ」感は増えた。「誰が」はまだ実機で要確認
- 更新直後にホームアイコンから開けない報告 → v46 で SW フォールバック修正

### Day 12（0719・続き）— 時間帯の会話差分

*体験*

*実施（v48）*

- 天候スパイスと併用（雨などドラマチックな日は両方寄り）

*判定*
- 台詞の時間は入ったが、**選択肢ラベルが時間固定のまま** → v49

### Day 12（0719・続き）— 選択肢の時間帯対応

*体験*
- 「おはよう」が夜に出ない。夕方なら「こんばんは」、夜の「夜更かし？」も朝なら「早起き？」など

*実施（v49）*
- `ITO_CHOICE_TIME_OVERRIDES` で表示ラベルを時間帯差し替え（内部キーは維持）

*判定*
- 夕なのに「夜風に当たる」が出る不整合 → v50 で時間ロック

### Day 12（0719・続き）— 用事ビートの時間ロック

*体験*
- 「夜風」は夜だけ。「朝の散歩／開店前」は朝だけ。夕は遠回り・手紙・信号などに寄る
- カードの「昼の長上着」などは **服の名前**（歩いた時間帯の色）で、いまの時刻表示とは別

*実施（v50）*
- `ITO_BEAT_TIME_LOCK` + 表示時にいまの帯へ差し替え

### Day 11（0711〜0712）— PiP・服・天気・空・GPS

日付をまたいだセッションを **Day 11** にまとめた。

*体験*
- 服屋の服が「どれも同じ柄」ではなくなった — 歩き方で **横帯／縦パネル／格子／やわらか／斜め** が変わる
